Detailed Description

Manages the valid set of skills (taxonomy) that the system recognizes.

Serves as the Single Source of Truth for:

  1. Skill Dictionary: What keywords constitute a "skill" (e.g., "Python", "Kubernetes").
  2. Grouping: Which category a skill belongs to (e.g., "Languages", "Cloud_Platforms").

This abstractions allows the underlying JSON/Python taxonomy file to change structure without breaking the rest of the pipeline.

Creates a reverse-lookup map to find the category of a given skill.

Iterates through the hierarchical TAXONOMY to build a flat dictionary: { "python": "Languages", "aws": "Cloud_Platforms", ... } This is essential for the GraphBuilder to assign group attributes to nodes.
